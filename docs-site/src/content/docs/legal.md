このラボで紹介している攻撃手法（コマンドインジェクション、リバースシェル、DB 窃取、バックドア設置、横展開スキャンなど）を、**許可のない外部の実サービスに対して実行すると、複数の日本の法律に同時に抵触** します。「1つの攻撃行為で複数の罪が併合的に成立する」のがサイバー犯罪の通例で、罰則も合算されていきます。

このドキュメントは、ラボの安全な遊び場と現実世界の境界線を引くための参考資料です。

> このページは法律の解説を目的としたものであって、法的助言ではありません。実際にトラブルに巻き込まれた場合は弁護士に相談してください。

## 1. 不正アクセス禁止法（不正アクセス行為の禁止等に関する法律）

正式名称は「不正アクセス行為の禁止等に関する法律」。**他人のシステムに対し、アクセス管理者の承諾なく制限を突破して侵入する行為そのもの** を禁じています。コマンドインジェクションは「認証を回避してサーバー側で任意のコマンドを実行させる」行為なので、第2条第4項に定める不正アクセス行為に該当します。

- **罰則**: 3年以下の懲役または100万円以下の罰金（第11条）
- **重要なポイント**: 「クライアントから依頼されたペネトレーションテストであっても、**システム所有者の許可がない第三者システムに対する診断は不正アクセスに該当する**」と明確に解釈されています。「セキュリティ研究目的」「善意で報告するつもりだった」も免責になりません

参考リンク:

- [総務省 国民のためのサイバーセキュリティサイト - 不正アクセス行為の禁止等に関する法律](https://www.soumu.go.jp/main_sosiki/cybersecurity/kokumin/basic/legal/09/)
- [警察庁 - 不正アクセス行為の禁止等に関する法律の解説（PDF）](https://www.npa.go.jp/bureau/cyber/pdf/1_kaisetsu.pdf)
- [e-Gov 法令検索 - 不正アクセス行為の禁止等に関する法律](https://laws.e-gov.go.jp/law/411AC0000000128)

## 2. 不正指令電磁的記録に関する罪（刑法168条の2、168条の3）

通称「ウイルス作成罪」。2011年の刑法改正で新設されました。**正当な理由なく、他人のコンピュータで意図に反する動作をさせる電磁的記録（プログラム等）を作成・提供・供用・取得・保管する行為** を禁じます。

ラボで紹介した攻撃に当てはめると：

- **リバースシェル用の bash 行**（`bash -i >& /dev/tcp/...`）を被害者サーバーで実行させる → **供用罪**
- **暗号通貨マイナーのスクリプトを `curl | sh` させる** → **供用罪**
- バックドアを書き込んで次回ログイン時に発火させる → **作成 + 供用罪**

罰則:

- 作成・提供罪: 3年以下の懲役または50万円以下の罰金（第168条の2第1項）
- 供用罪・取得・保管罪: 同上または2年以下の懲役・30万円以下の罰金

参考リンク:

- [警視庁 - 不正指令電磁的記録に関する罪](https://www.keishicho.metro.tokyo.lg.jp/kurashi/cyber/law/virus.html)
- [法務省 - いわゆるコンピュータ・ウイルスに関する罪について（PDF）](https://www.moj.go.jp/content/001267498.pdf)

## 3. 電子計算機損壊等業務妨害罪（刑法234条の2）

**他人の業務に使うコンピュータや電磁的記録を損壊したり、虚偽の情報・不正な指令を与えて、業務を妨害する罪** です。

ラボで紹介した攻撃に当てはめると：

- **DB ファイルを抜き出すために負荷をかけたり、ロックを乱す** → 該当しうる
- **ソースコードを改ざんしてアプリの動作を変える** → ど真ん中で該当
- **DDoS / 大量リクエストで停止させる** → 該当（警察庁が DDoS について明示的に注意喚起）
- **暗号通貨マイナーで CPU を専有してサービスを劣化させる** → 該当しうる

罰則: **5年以下の懲役または100万円以下の罰金**（不正アクセス禁止法より重い）。

参考リンク:

- [警察庁 - DDoS攻撃は犯罪です！](https://www.npa.go.jp/bureau/cyber/countermeasures/ddos_campaign.html)
- [e-Gov 法令検索 - 刑法 第234条の2](https://laws.e-gov.go.jp/law/140AC0000000045#Mp-Pa_2-Ch_35-At_234_2)

## 4. その他、同時に成立しうる罪

- **個人情報保護法** - DB から個人情報を抜けば、不正取得や漏えいに関する規定（第20条等）に違反。事業者として保管していたデータを抜けば組織側にも報告義務が発生します
- **不正競争防止法** - 営業秘密を抜き取ると営業秘密侵害罪（第21条）。10年以下の懲役または2000万円以下の罰金（個人）と非常に重い
- **著作権法** - サーバー上のソースコードや画像を取得・複製した場合、私的複製の例外を超えれば著作権侵害

## 「バグを見つけて報告するつもりだった」は通用しません

善意であっても **事前許可がなければ違法** です。脆弱性報告をしたい場合は次のような正規ルートを使ってください：

- **IPA / JPCERT/CC の脆弱性関連情報届出制度**（情報セキュリティ早期警戒パートナーシップ）: [https://www.ipa.go.jp/security/vuln/report/](https://www.ipa.go.jp/security/vuln/report/)
- **対象企業のバグバウンティ・プログラム**（HackerOne / Bugcrowd 等で公開されているスコープ内のみ）
- **対象企業の `security.txt` や脆弱性報告窓口**（許可された範囲・手段に従う）

スコープ外への積極的な攻撃テストは、いずれの正規制度でも **明示的に禁じられています**。

## 線引きのまとめ

技術的に可能なことと、法的にやっていいことは別です。攻撃者にとって「1日で DB を抜けるルート」は、自分が許可なく踏めば **不正アクセス禁止法 + ウイルス罪 + 業務妨害罪 + 個人情報保護法** が同時に成立するルートでもあります。

**ラボでは思いっきり試して、実環境では絶対にやらない**。この線を明確にした上で、攻撃の全体像を体験してください。
