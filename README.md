# AI Webapp Security ハンズオン 2026

Web アプリケーションのセキュリティ／パフォーマンスを **手を動かして学ぶ** ハンズオンリポジトリです。

各レクチャーのディレクトリには **意図的に問題が仕込まれた** Web アプリが入っています。アプリを起動して問題を体験し、コードを読んで原因を見つけ、修正するまでが各レクチャーの流れです。

セクション・レクチャーの一覧、技術スタック、起動方法などの詳細は [sections/README.md](./sections/README.md) を参照してください。

## 背景

AI にコードを書かせて Web アプリを作れる時代になりました。一方で、生成されたコードが本当に正しいか・安全かを判断する責任は人間に残ります。

- 出力されたコードが正しく動くか・安全かを判断するには、結局コードを読む力が必要です
- 効率的なプロンプトを書くにも、対象アプリの構造を把握できていないと、的外れな指示しか出せません
- チームで開発する場合は、自分の担当範囲がアプリ全体のどこに位置づくのかを説明できる必要があります

つまり、AI を使う前提でも（むしろ使うからこそ）、**アプリケーションを構造的に理解する力** が前提条件として求められます。

## 対象者

AI を使って Web アプリを作れるようにはなったが、生成されたコードの中身や安全性に不安がある初学者〜中級者を想定しています。

- AI が出したコードをそのまま使ってよいか自信が持てない
- 動いているように見えるが、本当に問題がないかを自分で確かめる手段を持っていない
- アプリの構造を言葉で説明することに不慣れ

そんな状態を抜け出したい人のための教材です。

## 目的

簡単な動作確認だけではわからない問題を見つける力を身につけることを目的としています。

そのために、**セキュリティとパフォーマンスの課題を意図的に持たせたアプリケーション** を題材として、

1. 不具合を体験する
2. コードを読んで原因を特定する
3. 修正して挙動が変わることを確認する

という流れを繰り返します。これを通して、表面的な動作確認の奥にある「アプリの構造」を捉える視点を養うことを目指します。

## ライセンス・注意事項

- 学習用リポジトリです。本番環境にデプロイしないでください
- 一部のレクチャーには **意図的にセキュリティ脆弱性** が含まれています
- 修正前のコードを実環境に流用するのは絶対に避けてください
