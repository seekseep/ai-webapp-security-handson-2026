# 03. インジェクション

インジェクション系の脆弱性は **OWASP Top 10 の常連** で、「ユーザー入力を文字列としてそのまま埋め込む」一行のコードから生まれます。共通する根本対策は **「入力値を構造ではなくデータとして扱う」** ことです。

## レクチャー

| レクチャー | テーマ |
|---|---|
| [01-sqli](./01-sqli/) | SQL インジェクション |
| [02-xss](./02-xss/) | XSS（Stored XSS） |
| [03-command-injection](./03-command-injection/) | コマンドインジェクション |

## このセクションのねらい

SQL のプレースホルダ（`?` バインド）、HTML 出力時のエスケープ、外部コマンドへの引数の安全な渡し方など、それぞれの場面でどんな仕組みが用意されているかを体感します。

> **注意**: このセクションのレクチャーで扱う攻撃手法は、**自分が用意した学習用環境** に対してのみ実行してください。許可のない外部システムに対する攻撃は **複数の日本の法律に同時に抵触** します。詳細は [LEGAL.md](../LEGAL.md) を必ず読んでから先に進んでください。
