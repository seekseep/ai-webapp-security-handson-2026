# 02. 認証・認可

Web アプリで最も発生しやすく、最も影響が大きい領域の一つが認証・認可です。「誰が」「何をしてよいか」の境界を曖昧にしたまま機能を作ると、ログインなしでデータが書き換わったり、他人のアカウントが乗っ取られたり、別の利用者の情報が覗けたりします。

## このセクションのねらい

典型的な3つの不備を題材に、次の観点を体験します。

- **認証の境界**: ログインが必要な処理とそうでない処理を、どこでどう分けるか
- **認証の強度**: パスワードの保管・照合の仕方ひとつでアカウントが守れるかどうかが決まる
- **認可の粒度**: ログインしているだけでなく「自分のものだけを操作できる」ことをどう保証するか

各レクチャー共通の起動手順と前提環境は [sections/README.md](../README.md) を参照してください。
