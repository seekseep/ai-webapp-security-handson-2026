## 学ぶこと

このハンズオン全体で扱うサンプルアプリ（ナレッジ共有ツール）の **環境構築の手順を体験** し、**アプリの全体像** を理解することがこの章のゴールです。

以降の章では、このアプリの各機能に意図的な問題（XSS, SQL インジェクション, 認可不備など）が仕込まれていきます。問題を見つけて修正するためには、**まずアプリの構造を知っておく** 必要があります。

具体的に身につけておきたいのは以下の3点です。

- **起動の流れ**：DB のテーブル作成 → シードデータ投入 → サーバー起動、という典型的な手順
- **アプリの機能**：記事・コメント・ユーザー・管理画面・ダッシュボードがどんな動きをするか
- **コードの地図**：ルーティング・DB アクセス・テンプレート・セッション管理がそれぞれどのファイルにあるか

## 説明

### TODO 1: アプリを起動する

[README.md](./README.md) の「起動方法」に手順がまとまっています。ローカル Node.js で動かす場合は次の通りです。

```bash
npm install
npm run database:init   # テーブル作成
npm run database:seed   # 初期データ投入（30ユーザー・200記事）
npm run dev             # 開発サーバー起動（ファイル変更で自動再起動）
```

ブラウザで http://localhost:3000 にアクセスして、記事一覧が表示されれば成功です。

Docker を使う場合は `docker compose watch` でサーバーが起動します。`init` と `seed` は別ターミナルで `docker compose exec app npm run database:init / :seed` を順に叩いてください（詳細は [README.md](./README.md) の「起動方法」を参照）。

### TODO 3: ディレクトリ構成とコードを覗く

このアプリは以下の構成で動いています。

- **Hono**（Web フレームワーク）：`app/server.js` でルートを登録し、各 `routes/*.js` がエンドポイントを実装
- **SQLite（better-sqlite3）**：`app/db.js` 経由で生 SQL を書く（ORM は使わない）
- **テンプレート**：`hono/html` のテンプレートリテラルでサーバーサイドレンダリング
- **セッション管理**：`app/middleware/session.js` で Cookie ベースのセッションを実装

実際にコードを開いて、次の問いに答えられるか確認してみてください。

1. **記事一覧の HTML はどこで組み立てている？**
   → ヒント: `app/routes/articles.js` の `app.get('/')` ハンドラ
2. **ログイン状態はどう判定している？**
   → ヒント: `app/middleware/auth.js` の `currentUser()` ミドルウェア
3. **DB へのアクセスはどう書かれている？**
   → ヒント: `db.prepare('...').get(...)` / `.run(...)` / `.all(...)` の形でルートから直接呼ばれている
4. **共通レイアウト（ヘッダー・ナビ）はどこ？**
   → ヒント: `app/components/layout.js`

ファイルの対応関係の全体像は [README.md](./README.md) の「ディレクトリ構成」セクションにまとめてあります。
